# Avatar-poc

Single-command prototype for generating:

- a speech `.wav` from input text
- a SadTalker `.mp4` from one input image or video plus that WAV

The SadTalker leg now runs against a local SadTalker checkout. Avatar does not modify the SadTalker codebase.

## Prerequisites

- Python 3.9+ for this repo
- A local SadTalker checkout with its dependencies installed
- SadTalker checkpoints downloaded locally

Export the SadTalker paths before running:

```bash
export SADTALKER_DIR=/Users/yevhen/PycharmProjects/SadTalker
export SADTALKER_CHECKPOINT_DIR=/Users/yevhen/PycharmProjects/SadTalker/checkpoints
```

If SadTalker needs a different Python environment than this repo, also set:

```bash
export SADTALKER_PYTHON=/path/to/your/sadtalker/python
```

## Usage

Accelerated mode, which lets SadTalker use CUDA when available and otherwise falls back to CPU:

```bash
python3 run.py \
  --input samples/input/example-photo.png \
  --text "Hello from the Avatar prototype. This text is spoken by TTS model. The video was generated by SadTalker. Have a nice day" \
  --runtime cpu-gpu
```

CPU-only run:

```bash
python3 run.py \
  --input samples/input/example-photo.png \
  --text "Hello from the Avatar prototype. This text is spoken by TTS model. The video was generated by SadTalker. Have a nice day" \
  --runtime cpu
```

If you already have a WAV and just want to run the SadTalker leg:

```bash
python3 run.py \
  --input samples/input/women-video-7sec.mp4 \
  --audio path/to/input.wav \
  --runtime cpu-gpu
```

Outputs are written to `samples/output/` by default as:

- `<stem>.wav`
- `<stem>.mp4`

## Notes

- `--input` accepts either an image or a video file. With video input, Avatar passes the file straight to SadTalker, which uses the first frame as the reference source.
- `--runtime cpu` passes SadTalker's native `--cpu` flag.
- `--runtime cpu-gpu` leaves SadTalker in its default mode so it can use CUDA when that local environment supports it.
- `--sadtalker-dir`, `--checkpoint-dir`, and `--sadtalker-python` override the corresponding environment variables.
- If `--sadtalker-python` is not set, Avatar tries a local SadTalker virtualenv first and then common conda env locations such as `~/miniconda3/envs/sadtalker/bin/python`.
- `--preprocess full --still` is a useful combination for full-frame inputs.
- `--size` supports SadTalker's 256 and 512 face models.
- You can skip TTS entirely with `--audio <existing.wav>`.
- You can keep SadTalker's temporary directory with `--keep-sadtalker-temp`.
