# Avatar-poc

Single-command prototype for generating:

- a speech `.wav` from input text
- an EchoMimic `.mp4` from one input image or video plus that WAV
- a final `.mp4` with RobustVideoMatting background removal after EchoMimic

EchoMimic runs against a local checkout and conda environment. RobustVideoMatting
is loaded from the official TorchHub repository on demand.

## Prerequisites

- Python 3.9+ for this repo
- `pip install -r requirements.txt` in this repo's Python environment
- A local EchoMimic checkout with its dependencies installed
- A local EchoMimic `pretrained_weights` checkout with the actual Git LFS model files downloaded

Export the EchoMimic paths before running:

```bash
export ECHOMIMIC_DIR=/Users/yevhen/PycharmProjects/EchoMimic
export ECHOMIMIC_PYTHON=/Users/yevhen/miniconda3/envs/echomimic/bin/python
```

If your `pretrained_weights` repo still contains 100-150 byte pointer files, pull
the actual checkpoints before running:

```bash
git -C /Users/yevhen/PycharmProjects/EchoMimic/pretrained_weights lfs pull
```

## Usage

```bash
python3 run.py \
  --input samples/input/me.jpg \
  --voice am_echo \
  --text "Hello from the Avatar prototype. This text is spoken by TTS model. The video was generated by EchoMimic. Have a nice day"
```

If you already have a WAV and want to run the full pipeline:

```bash
python3 run.py \
  --input samples/input/me.jpg \
  --audio samples/output/am_echo.wav
```

If you already have a generated talking-head video and want to start at the
background stage, skipping EchoMimic:

```bash
python3 run.py \
  --input samples/output/raw-echomimic/video.mp4 \
  --audio samples/output/raw-echomimic/sound.wav \
  --start-stage background
```

Outputs are written to `samples/output/` by default as:

- `<stem>.wav`
- `<stem>.mp4`

## Notes

- `--input` accepts either an image or a video file. With video input, Avatar extracts the first frame and uses that image as EchoMimic's reference source.
- `--start-stage background` requires a video input because RobustVideoMatting is a video-matting stage.
- `--echomimic-dir` and `--echomimic-python` override the corresponding environment variables.
- If `--echomimic-python` is not set, Avatar tries a local EchoMimic virtualenv first and then common conda env locations such as `~/miniconda3/envs/echomimic/bin/python`.
- `--echomimic-variant accelerated` is the default. Use `--echomimic-variant standard` if you want the upstream non-accelerated audio-driven script.
- `--device cpu` is the default, which avoids the upstream CUDA-only assumption on macOS. You can still pass `--device mps` or `--device auto` if your EchoMimic environment supports them.
- You can skip TTS entirely with `--audio <existing.wav>`.
