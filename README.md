# Avatar-poc

Single-command prototype for generating:

- a speech `.wav` from input text
- a SadTalker `.mp4` from one input portrait image plus that WAV

The SadTalker leg now follows the Docker flow documented in SadTalker's install guide by invoking the community image directly: [`wawa9000/sadtalker`](https://hub.docker.com/repository/docker/wawa9000/sadtalker).

## Prerequisites

- Python 3.9+ for this repo
- Docker installed and running
- SadTalker image pulled locally:

```bash
docker pull wawa9000/sadtalker
```

## Usage

GPU-enabled host, matching SadTalker's Docker example:

```bash
python3.11 run.py \
  --input samples/input/women-video-7sec.mp4 \
  --text "Hello from the Avatar prototype. This text is spoken by TTS model. The video was generated by SadTalker. Have a nice day" \
  --docker-gpus all
```

CPU-only or Apple Silicon Docker run:

```bash
python3 run.py \
  --input samples/input/example-photo.png \
  --text "Hello from the Avatar prototype. This text is spoken by TTS model. The video was generated by SadTalker. Have a nice day" \
  --cpu
```

If you already have a WAV and just want to run the SadTalker leg:

```bash
python3 run.py \
  --input samples/input/example-photo.png \
  --audio path/to/input.wav \
  --docker-gpus all
```

Outputs are written to `samples/output/` by default as:

- `<stem>.wav`
- `<stem>.mp4`

## Notes

- No local SadTalker checkout, checkpoints, or conda env are required anymore.
- `--docker-image` defaults to `wawa9000/sadtalker`. Override it or `SADTALKER_DOCKER_IMAGE` if you want a different image tag.
- `--docker-executable` defaults to `DOCKER_EXE` or `docker` on your `PATH`.
- `--docker-gpus all` mirrors the SadTalker Docker docs. Leave it unset, or use `--cpu`, on machines without NVIDIA Docker support.
- `--docker-platform` defaults to `linux/amd64` on Apple Silicon because this SadTalker image is amd64-only.
- This repo mounts a tiny compatibility entrypoint into the container so `--cpu` works with the current `wawa9000/sadtalker` image on non-NVIDIA machines.
- Dockerized SadTalker does not use Apple MPS inside the container, so Apple Silicon will run this path on CPU through Docker emulation.
- TTS defaults to `--tts-engine auto`, which tries Kokoro first and falls back to macOS `say`.
- You can force the fallback path with `--tts-engine macos_say`.
- You can skip TTS entirely with `--audio <existing.wav>`.
- You can keep SadTalker's temporary directory with `--keep-sadtalker-temp`.
